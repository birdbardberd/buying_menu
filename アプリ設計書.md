# 買い物リストアプリ 仕様書・設計書

## 1. アプリ概要

過去の献立（レシピセット）を選択するだけで、スーパーの売り場カテゴリごとに自動整列された買い物リストが生成される、週一まとめ買い派のための支援アプリ。

---

## 2. ターゲットユーザー

* 週に一度まとめて食料品を買い出しに行く主婦・主夫
* 献立を考える手間を減らし、買い忘れを防ぎたい人

---

## 3. 主要機能 (Feature)

### ① 献立マスター登録機能

* 献立名（例：肉じゃが）と、それに必要な材料・カテゴリをセットで登録。
* 一度登録した献立はアーカイブされ、何度でも再利用可能。

### ② 献立選択（買い物リスト生成）

* 登録済みの献立リストから、今回作るものにチェックを入れる。
* チェックした献立に含まれる材料が、自動的に「買い物リスト」へマージされる。

### ③ カテゴリ別自動整列リスト

* 追加された材料を、設定されたカテゴリ（野菜、肉、魚など）ごとに自動で仕分け表示。
* チェックボックスをタップすることで「カゴ入れ完了」状態にする。

---

## 4. カテゴリ定義 (Data Structure)

アプリ内で使用する基本カテゴリ階層は以下の通り。

| 大カテゴリ | 小カテゴリ（例） |
| --- | --- |
| **野菜** | キャベツ、人参、じゃがいも |
| **肉** | 肉団子、豚肉、サラダチキン、鶏肉 |
| **魚** | 鮭、刺身、切り身 |
| **肉・魚以外の冷蔵** | 卵、納豆、豆腐、うどん、惣菜 |
| **常温保存** | お菓子、鍋の素、味の素シリーズ |
| **その他** | 調味料、日用品 |

## 5. 画面設計 (UI Design)

### 画面 A：献立選択画面 (Menu Selection)

* **上部:** 「今週の献立をえらぶ」タイトル
* **中央:** 献立カードのリスト（スクロール可能）
* カード内：献立名、主な材料のプレビュー、選択チェックボックス


* **下部:** 「新しい献立を追加する」ボタン、タブバー

### 画面 B：買い物リスト画面 (Shopping List)

* **上部:** 「買い物リスト」タイトル
* **中央:** カテゴリ別の材料リスト
* カテゴリ見出し（パステルカラー背景）
* 材料名（チェックボックス付き）


* **下部:** タブバー（献立、買い物、履歴、設定）

---

## 6. 技術スタック案（参考）

* **フロントエンド:** React Native または Flutter（iOS/Android両対応のため）
* **状態管理:** 選択された献立情報を保持するストア
* **データ保存:** ローカルDB（SQLite / Realm）または Firebase（クラウド同期する場合）

---

## 7. デザインガイドライン

* **テーマカラー:** パステルカラー（イエロー、グリーン、ピンク、ブルー）
* **文字:** 黒色（視認性重視）
* **スタイル:** 角丸（Radius: 12px〜）を多用した、柔らかく温かみのあるUI
